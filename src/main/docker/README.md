# Docker Files — Not Currently Used

These four Dockerfiles were auto-generated by `quarkus create app` and are
kept intentionally, even though CloudAudit deploys to AWS Lambda, not to a
standalone Docker container.

- Dockerfile.jvm          — runs the app as a compiled JAR on a JVM
- Dockerfile.legacy-jar   — older JAR packaging style
- Dockerfile.native       — GraalVM native compilation (fast startup, slow build)
- Dockerfile.native-micro — native build on a minimal base image

Kept for reference / in case the deployment target ever changes (e.g. if a
non-Lambda containerized version were ever needed). Not part of the current
Terraform deployment, not referenced by any CI/CD step, and not required for
local development — Quarkus Dev Services handles local testing separately.
